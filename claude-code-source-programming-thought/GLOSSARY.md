# 术语表

## 基础术语

- `REPL`：Claude Code 的交互壳层，不只是输入框，而是会话、状态、工具确认、任务视图共同组成的运行前台。
- `回合`：一次从用户输入开始，到模型回复、工具调用、工具结果回填、状态更新结束的闭环。
- `消息`：系统、用户、助手、进度、工具结果等统一在消息流中流动，是 Claude Code 的底层数据介质。
- `主循环`：由 `query` 驱动的回合执行核心，负责串起模型请求、工具执行、恢复策略与上下文治理。

## 运行时术语

- `Tool`：模型可以调用的能力单元，既有输入输出 schema，也有权限、并发和结果映射规则。
- `Tool Pool`：当前会话可见工具的集合，不是固定清单，而是由运行时环境、特性开关和上下文共同装配。
- `Tool Orchestration`：对工具调用进行分组、并发执行、串行执行、上下文回写的调度过程。
- `Plan Mode`：一种显式工程协议，要求先探索和设计，再请求批准后进入实现。
- `Permission Context`：权限模式和决策上下文的集合，决定某个工具调用是自动放行、询问、拒绝还是升级。

## 多代理术语

- `Agent`：可独立执行一段上下文与工具集的代理单元。
- `Subagent`：由主代理派生出来、承担局部任务的代理。
- `Team Agent`：共享任务与协作协议的一组代理，重点不在“多开”，而在“协同”。
- `Mailbox`：代理之间传递消息、权限响应、空闲通知的通信机制。
- `Task List`：团队共享的任务台账，用于认领、更新、阻塞关系和收敛。

## 上下文术语

- `System Prompt`：系统层指令集合，不是单个字符串，而是多层拼装结果。
- `Memory`：用户、项目、团队等维度的可注入上下文集合。
- `Compact`：在上下文预算受限时对历史进行压缩、折叠或清理的策略。
- `Token Budget`：回合与会话在上下文和输出上的预算约束。

## 扩展术语

- `Skill`：以 Markdown/frontmatter 声明的可调度 prompt 程序。
- `Command`：Slash command 及其扩展体系的统一运行时表示。
- `Plugin`：提供命令、技能和其他扩展组件的容器。
- `MCP`：Model Context Protocol，在这里承担远程能力总线角色，而不只是工具桥接。
