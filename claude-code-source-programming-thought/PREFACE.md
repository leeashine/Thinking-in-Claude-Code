# 前言与阅读指南

状态：`completed`

这本书不是 Claude Code 的功能手册，也不是按目录顺序解释源码的导游文档。它更接近《JAVA编程思想》那种写法：借一套真实系统的源码，讨论一组更稳定的工程问题。Claude Code 之所以适合这样阅读，不是因为它“功能很多”，而是因为它同时处理了 REPL、主循环、工具 ABI、权限、Plan Mode、子 agent、团队协作、记忆、compact、远程能力、观测与恢复这些现代 Agent 系统绕不开的问题。

本书只使用当前工作目录下 `src/` 的源码快照，不追溯历史提交，也不引入未核实的外部资料。换句话说，这不是关于“Claude Code 官方理念”的书，而是关于“从当前源码里能被验证出来的系统设计”的书。任何看似宏大的结论，如果不能回落到实现、调用链或注释，本书都不会把它写成事实。

全书建议按卷顺序阅读。卷一到卷三先建立运行时主线，从 REPL、query loop 到 team agent；卷四转向上下文系统，理解 prompt、memory 与 compact 如何共同塑造智能；卷五再看扩展与边界，把命令、技能、插件、MCP 和远程会话放在一起；卷六最后收束到工程化与方法论，讨论为什么 Claude Code 可以在复杂约束下仍然持续演进。

如果你的兴趣是复刻类似系统，而不是完整阅读 Claude Code，可以按下面的最短路径走：

1. 先读 `CH03`、`CH04`、`CH05`，建立主循环、工具 ABI 和权限边界的认识。
2. 再读 `CH07`、`CH08`、`CH09`，理解子 agent 与共享任务状态。
3. 接着读 `CH10`、`CH11`、`CH12`，理解上下文治理。
4. 最后读 `CH17`、`CH18`，把局部机制抽象成可迁移方法。

本书中的“思想”并不意味着玄学总结。这里所谓“思想”，指的是那些已经被源码结构化表达出来、并且足以迁移到其他工程中的判断。例如，为什么权限是一条运行时管线，为什么 compact 不是摘要按钮，为什么 team agent 需要共享 task list，为什么 memory 不能只做全量前缀注入。只要你把这些判断迁移到自己的系统里，这本书就发挥了它应有的价值。
