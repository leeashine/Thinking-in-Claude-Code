# 序言

## 为什么学习Claude Code源码

在人工智能时代，编程工具正在经历一场深刻的变革。Claude Code作为Anthropic官方推出的AI编程助手，其源代码为我们展示了如何构建一个现代化的、工具驱动的AI Agent系统。

### 工具驱动的AI Agent范式

传统的AI应用往往将模型视为"黑盒"，通过简单的API调用获取响应。Claude Code展示了另一种可能：将AI Agent构建为一个**工具驱动系统**。

在这个系统中：
- **模型是决策者**：根据上下文选择合适的工具
- **工具是能力扩展**：每个工具赋予Agent特定的能力
- **状态是上下文**：维护会话的完整历史和当前状态
- **任务是执行单元**：异步处理各种后台操作

这种架构使得Agent的能力可以无限扩展——只需添加新的工具。

### CLI作为AI Agent的最佳载体

为什么Claude Code选择命令行界面（CLI）而非图形界面（GUI）？

1. **效率优先**：CLI是程序员的自然工作环境
2. **管道组合**：可以与其他命令行工具无缝集成
3. **自动化友好**：易于脚本化和自动化
4. **资源高效**：终端渲染开销远小于GUI
5. **键盘驱动**：程序员习惯键盘操作

Claude Code通过Ink框架实现了React式的终端UI，让开发者可以用熟悉的组件模型构建CLI应用。

### 终端UI的复兴与创新

Ink框架将React的声明式编程模型引入终端，这是一个创新：
- 组件化开发：UI拆分为可复用的组件
- 状态驱动渲染：UI = f(state)
- Flexbox布局：使用CSS Flexbox概念布局终端
- 增量更新：只更新变化的部分，减少闪烁

### 如何阅读本书

**第一遍：快速浏览**
- 阅读序言和第一部分核心架构
- 了解整体设计思想和架构概览
- 跳过复杂的代码细节

**第二遍：深入实践**
- 按顺序阅读各章节
- 对照源码理解每个概念
- 尝试修改和实验

**第三遍：专题研究**
- 选择感兴趣的章节深入
- 结合附录的设计模式总结
- 应用到自己的项目中

### 源码学习方法

本书基于`src_glm`版本的Claude Code源码分析。建议：

1. **使用本书提供的文件路径**快速定位源码
2. **使用grep/ripgrep**搜索相关概念
3. **使用VSCode或Zed**等编辑器进行代码跳转
4. **实际运行Claude Code**观察行为

### 实践建议

- 不要只读不动手
- 尝试添加自己的工具
- 尝试修改现有组件
- 尝试扩展任务系统
- 将所学应用到自己的项目
